06 — Foundations

Motion

One hover grammar, applied everywhere: image zoom, headline underline, and a top-down background fill. Always paired with hover:none and reduced-motion resets.

Image zoom
scale(1.05) features · scale(1.08) thumbs
transform .35s ease · cropped by sharp frame
Headline underline
underline · offset 3px · thickness 1.5px
heading only, on card hover
Progressive fill
::before height 0 → 100% · .35s ease
#F6F7F9 (light) · #EFE8DA (events band)
Color transitions
background / border / color · .2s ease
dark items → #13203A · play badge → orange
Poll fill
width 0 → N% · .8s cubic-bezier(.22,.61,.36,1)
pct fades in with .15s delay
Focus & a11y
focus-visible: 2px blue outline, offset 2px
@media hover:none + prefers-reduced-motion resets
07 — Governance

Do / Don't

The rules that make output look like TechCircle rather than a generic news site.

Do
  • Keep every corner sharp — radius 0 on cards, buttons, inputs, image frames.
  • Set all headings in Young Serif at weight 400.
  • Use IBM Plex Mono for every piece of metadata, badge, and utility button.
  • Divide with 1px #E0E2E7 hairlines; break the page with tinted bands.
  • Use the 3px orange bar to anchor every section head.
  • Apply the full hover grammar: zoom + underline + progressive fill.
Don't
  • No rounded corners, pill buttons, or drop-shadows on cards.
  • No bold serif headings; no Young Serif in body copy or buttons.
  • No orange as a large background — accent only; use #C47B14 for orange text on white.
  • No gradients except image shades and the dark panel.
  • No blue for anything except interaction (authors, hovers, focus).
  • Don't replace the hover grammar with opacity fades or lift shadows.
